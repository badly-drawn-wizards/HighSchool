Designing a shader

FancyShader uses GLSL. The GLSL in FancyShader describes a program that runs for each pixel on the preview canvas with the pixel coordinate as the input and the pixel color as the output. For example the GLSL program:

uniform vec2 resolution;
void main() {
    // normalize the coordinates from pixel dimensions to unit dimensions
    vec2 position = gl_FragCoord.xy/resolution;
    // GLSL colors are represented as 4-dimensional vectors containing red, blue, green and opacity values ranging from 0.0 to 1.0
    gl_FragColor = vec4(position.x);
}

is a grayscale gradient that goes from black to white as one goes from left to right. This is a public demo Example 1 (Greyscale Gradient). One can see that gl_FragCoord was not the only input. The uniform resolution was also inputted, which is a vector containing the preview canvas pixel dimensions. Another such uniform provided by FancyShader is time which is a float rather than a vec2. This allows for animation to be introduced.

If the code entered is invalid and does not compile, the error shader will be shown. This can be seen in the public demo Example 2 (Error Shader)
